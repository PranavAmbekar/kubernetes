Pods

Smallest deployable units in Kubernetes

Basic Operations

kubectl get pods
List all pods in current namespace
kubectl get pods -n <namespace>
List all pods in specific namespace
kubectl get pods --all-namespaces
List all pods in all namespaces
kubectl get pods -A
List all pods in all namespaces (shorthand)
kubectl get pod <pod-name> -o yaml
Get pod details in YAML format
kubectl get pod <pod-name> -n <namespace> -o yaml
Get pod details in YAML format from specific namespace
kubectl describe pod <pod-name>
Show detailed pod information
kubectl describe pod <pod-name> -n <namespace>
Show detailed pod information from specific namespace
kubectl delete pod <pod-name>
Delete a specific pod
kubectl delete pod <pod-name> -n <namespace>
Delete a pod from specific namespace
kubectl logs <pod-name>
View pod logs
kubectl logs <pod-name> -n <namespace>
View pod logs from specific namespace
kubectl logs <pod-name> -c <container>
View logs from specific container
kubectl logs -f <pod-name>
Stream pod logs in real-time
kubectl logs -f <pod-name> -n <namespace>
Stream pod logs from specific namespace
kubectl exec -it <pod-name> -- /bin/bash
Get shell access to pod
kubectl exec -it <pod-name> -n <namespace> -- /bin/bash
Get shell access to pod in specific namespace
kubectl port-forward <pod-name> 8080:80
Forward local port to pod

üí° Pro Tips

  • Use -o wide flag for more detailed output
  • Add --watch to monitor resource changes in real-time
  • Use kubectl explain <resource> to get field descriptions
  • Set alias: alias k=kubectl for faster typing
  • Use kubectl api-resources to list all available resources
  • Add --dry-run=client -o yaml to generate YAML without creating resources
  • Use -n <namespace> to work with resources in specific namespaces
  • Use -A or --all-namespaces to work across all namespaces
  • Set default namespace: kubectl config set-context --current --namespace=<name>
